codelite-debugger-fix

   This fix for the codelite debugger works on windows. Build, clean and run worked fine right from the start.  The problem cropped up when trying to use the debugger.  

When trying to debug I kept getting the following error: 

'Debugger exited with the following error string: "During startup program exited with code 0xc0000139." '

Meaning: The linker could not access the libgcc & libstdc++ libraries. I set the path for the g++.exe in my environments paths but, that did not fix the issue.

   One suggestion was to set the flags -static-libgcc -static-libstdc++ on the default project template and save it. This works for projects created on my system but, not for projects imported into codelite from exernal sources.

The Fix:

Navagate to: Setting > build settings > tools > linker path and edit the linker path as follows.

Change linker path from:  <installed drive>:/<install folder>/bin/g++.exe

To: <installed drive>:/<install folder>/bin/g++.exe -static-libgcc -static-libstdc++

(Note: replace <installed drive> with your drive letter & <install folder> with the folder your compiler is installed in.)

Save and restart Codelite.

This will make the debugger error go away and work for locally created projects and imported projects without any more hassles.

There you go....One major hurdle out of the way....And it only took a couple hours of research and 1 minute to fix with an edit :( but, Hey atleast it works now :)
